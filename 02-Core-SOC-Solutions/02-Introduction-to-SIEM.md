
# Introduction to Security Information and Event Management (SIEM)

## Overview

Security Information and Event Management (SIEM) is one of the core technologies used in a Security Operations Center (SOC). It centralizes security logs from multiple systems, correlates events, detects suspicious activity, and provides analysts with a single platform for monitoring and investigating security incidents.

Instead of manually checking logs from individual devices, analysts use a SIEM to search, correlate, and investigate events across the entire environment.

# Why SIEM is Important

Modern organizations generate millions of log events every day from servers, endpoints, network devices, cloud services, and security tools.

Without a centralized platform, monitoring these logs would be nearly impossible.

A SIEM enables analysts to:

- Collect logs from multiple sources
- Normalize different log formats
- Correlate related events
- Generate alerts
- Investigate incidents efficiently
- Support compliance and auditing

---

# How a SIEM Works

A simplified SIEM workflow:

1. Devices generate security logs.
2. Logs are forwarded to the SIEM.
3. The SIEM normalizes and indexes the data.
4. Correlation rules analyze incoming events.
5. Suspicious activity generates alerts.
6. SOC analysts investigate the alerts.
7. Findings are documented and escalated if necessary.

# Key SIEM Functions

## Log Collection

Collects security events from multiple devices into one platform.

---

## Log Normalization

Converts different log formats into a common structure, making analysis easier.

---

## Event Correlation

Identifies relationships between multiple events.

Example:

- VPN login
- Privilege escalation
- PowerShell execution

Individually these may appear harmless.

Together they may indicate an attacker.

---

## Alerting

Correlation rules generate alerts when suspicious patterns are detected.

Example:

- Multiple failed logins
- Impossible travel
- Malware detection
- Privileged account misuse

---

## Investigation

Analysts use the SIEM to:

- Search logs
- Build timelines
- Identify affected systems
- Find Indicators of Compromise (IOCs)

# SIEM vs EDR

| SIEM | EDR |
|------|-----|
| Collects logs from many systems | Monitors endpoint devices |
| Centralized visibility | Endpoint visibility |
| Correlates events | Detects endpoint behaviour |
| Supports organization-wide investigations | Supports endpoint investigations |

Both technologies complement each other and are commonly used together.

---

# Real-World Example

An attacker successfully guesses a user's VPN password.

The SIEM receives:

- VPN authentication logs
- Active Directory login events
- Windows Event Logs
- EDR telemetry

A correlation rule identifies unusual login behavior followed by suspicious PowerShell execution.

The SIEM generates a high-severity alert, allowing the SOC analyst to investigate the incident before further compromise occurs.

# Interview Questions

## What is a SIEM?

A SIEM is a centralized security platform that collects, stores, correlates, and analyzes logs from multiple systems to help security analysts detect and investigate threats.

---

## Why is event correlation important?

Individual events may appear harmless on their own. Correlation combines related events to identify suspicious patterns that could indicate an attack.

---

## What types of logs are commonly collected by a SIEM?

Examples include:

- Windows Event Logs
- Linux Syslog
- Firewall logs
- VPN logs
- Active Directory logs
- IDS/IPS alerts
- EDR telemetry
- Cloud service logs

---

## Does a SIEM replace EDR?

No.

A SIEM provides centralized visibility across many systems, while an EDR focuses on monitoring and investigating endpoint devices. Organizations commonly use both technologies together.
