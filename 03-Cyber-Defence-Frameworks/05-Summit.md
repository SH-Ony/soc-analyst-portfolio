# Summit

## Overview

Summit is a practical challenge room designed to reinforce the concepts learned throughout the Cyber Defence Frameworks module.

Instead of introducing new frameworks, this room requires the learner to analyze attacker activity, interpret security information, and apply defensive frameworks such as MITRE ATT&CK, the Cyber Kill Chain, and the Pyramid of Pain to understand an attack.

The focus is on analytical thinking rather than memorizing theory.

---

# Learning Objectives

After completing this room, I can:

- Apply defensive security frameworks to real-world scenarios.
- Analyze attacker behaviour during an investigation.
- Identify Indicators of Compromise (IOCs).
- Relate attacker actions to MITRE ATT&CK tactics and techniques.
- Develop a structured approach to security investigations.

---

# Investigation Process

During this challenge I practiced following a structured investigation process:

1. Review available evidence.
2. Identify suspicious activity.
3. Collect Indicators of Compromise (IOCs).
4. Map attacker behaviour to defensive frameworks.
5. Build an attack timeline.
6. Draw conclusions based on evidence.

Rather than jumping directly to conclusions, each finding should be supported by observable evidence.

---

# Frameworks Applied

Throughout the investigation I used concepts from:

- MITRE ATT&CK
- Cyber Kill Chain
- Unified Kill Chain
- Pyramid of Pain

Each framework provides a different perspective:

- MITRE ATT&CK explains attacker behaviour.
- Cyber Kill Chain identifies the attack stage.
- Unified Kill Chain provides a broader enterprise view.
- Pyramid of Pain helps prioritize defensive improvements.

---

# SOC Analyst Workflow

A typical workflow during an investigation includes:

1. Receive an alert.
2. Review available logs.
3. Identify affected systems.
4. Determine attacker activity.
5. Classify the incident.
6. Document findings.
7. Escalate if necessary.

This room reinforces the investigative mindset expected from a Level 1 SOC analyst.

# Key Concepts Reinforced

The challenge highlighted several important ideas:

- Security investigations should always be evidence-based.
- Multiple data sources provide a clearer understanding of incidents.

# Real-World Application

In a Security Operations Center, analysts rarely investigate alerts using a single tool or framework.

Instead, they combine:

- SIEM alerts
- Endpoint telemetry
- Threat intelligence
- Security frameworks
- Organizational procedures

This room reflects that investigative approach by requiring information from multiple sources to reach a conclusion.

# Interview Questions

## Why are practical investigation exercises important?

They develop analytical thinking and help analysts apply theoretical knowledge to realistic security incidents.

---

## Why should investigations be evidence-based?

Evidence-based investigations reduce assumptions and ensure conclusions are supported by observable facts.

---

## Why use multiple frameworks during an investigation?

Different frameworks provide different perspectives on the same incident, helping analysts better understand attacker behaviour, attack progression, and defensive opportunities.
